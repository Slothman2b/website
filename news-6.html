Rhino Linux 2023.1, how we made the distro.

Rhino Linux has now officially moved out of Beta! We have released Rhino Linux 2023.1 on x86_64, ARM, Pine64 and Raspberry Pi devices.

Quite a lot has changed since our first beta release back in April 2023, and so instead of a generic changelog, I want to do something a little different and discuss what actually went into the making of the operating system.

You can download Rhino Linux 2023.1 here.
Contact us here.
Or maybe donate to us here.

The concept

The concept of the Operating System came from my (AJ) previous project. Rolling Rhino Remix was a quick little hobby project I slapped together with little to no care for a bit of fun, however people started using it. This became unmaintainable in the long run and so in October 2022 I came up with a new vision for how I wanted my desktop operating system to go.

The first little idea came from when I saw VanillaOS' setup utility, and wanted to include that into Rolling Rhino Remix. From there the idea expanded into making a distribution based entirely on Pacstall.

Still we had one issue, GNOME was far too large to package into pacstall with our smaller team, and I wanted this operating system to have a more stable, and customisable GTK desktop environment. So we went with XFCE instead.

The plan at this stage was to still use the name "Rolling Rhino Remix", but scrap older ISOs and force users to upgrade to the new one after 3 months. This was not a good strategy and later we decided that changing it to a successor distribution would be a better fit.

After that I began work on branding, and we eventually settled on the name "Rhino Linux"

The builder

To be honest, I had little-to-no involvement with the builder, and so I'll hand over to cat-master21, the original creator for the Rhino Linux ISO builder, and Oklopfer our system's lead to discuss more about this.

Cat-master21: Rhino Linux's image building system uses a tool called LiveBuild created by Debian with some shell scripts for automation which we utilise from VanillaOS, which had forked it from elsewhere.

With the creation of Rhino Linux it would be best to switch from our old system in Rolling Rhino Remix that essentially unmounted and remounted an ISO with changes, this had it's various disadvantages and was dependent on Ubuntu's ISO structure.

Now the image building system creates everything from the ground-up so it is not dependent on that also removing the need to download Ubuntu's image speeding things up.

Package management

Hey! It's AJ again, One of the fundamental ideas of the distribution was to unify all methods of Package management, so users aren't running 5 commands to update their system, or find an app they want. This is where the idea for rhino-pkg came from. It's a bash script wrapper around Pacstall, Apt, Snap and Flatpak that allow for you to easily search and install applications as well as update your system.

The XFCE Desktop

Originally the plan was to utilise an XFCE Desktop with a similar layout to KDE Plasma or Microsoft Windows. Overall the user experience of this worked well, however we later decided to go a different route as we will discuss later on in this post.

Our first beta release

"Rhino Linux is pleased to announce that it has entered open beta. This is the collective work of 5 months of active development from the Rhino Linux team and other contributors." - This was the first line in the blog post where I announced the open beta of Rhino Linux, and suddenly we had a handful of downloads, and our first bug reports.

Greater user experience

I was at my girlfriend's for a while, with no laptop and no way to update my website, so I asked my girlfriend if I could borrow her M1 Macbook Pro. I also realised that it was out of date, by one major release and so updated it for her. I was impressed by how simple and painless the upgrades were. Nothing compared to my experience on Linux, and so I gained the idea to create rhino-system, I just needed to find a developer skilled enough to execute it.

The idea of macOS' user experience became stuck in my head, and also led to the development of the Unicorn Desktop and other features that some may see was inspired by Apple, such as RhinoDrop.

The Unicorn Desktop

Inspired by designs such as macOS, Pop!_OS COSMIC and GNOME we crafted Unicorn. Originally supposed to be an independent desktop environment based on xfwm4-wayland it would've utilised a collection of free software components modified to fit our evolving design language.

In time we refined the concept of Unicorn to be a desktop experience based on XFCE, bringing XFCE into a more modern age.

Your System

We ended up finding a talented gtk4-libadwaita developer, Axtloss. She began work on rhino-system, Also known as the "Your System" app. It was inspired by "About this mac" found on macOS. It is a GTK graphical application that displays system information and allows for you to update your system graphically. It was planned to be included in our fifth beta release, however there was an issue...

Big trouble in little Pacstall

Hey, Elsie, developer of Pacstall, here. An issue we had during this entire process was with pacstall packages depending on other pacstall packages. When users went to upgrade, they would be upgraded in alphabetical order, even if the main package was installing before its dependencies. This would cause issues with apt, and was not workable long-term. So I created a system that would essentially drop packages into "buckets" based on its traits as a package, such as if it had dependencies, pacstall dependencies, etc. What this would allow me to do is sort packages into a corrected order, so that dependants would be installed before the main package. Now rhino packages can be installed in the correct order, along with other programs that may need this.

Cleaning up

After beta6 and beta7, we had a few small issues to clean up, and from there we have been able to release Rhino Linux 2023.1.

We also had to work closely with Elsie to time the release with Pacstall 4.0.0, ensuring that there are little-to-no issues for users upgrading from previous versions of Rhino Linux to the full release.

I want to thank everyone who has supported our distribution, either by joining our community, utilising it, spreading our message or donating to us on Kofi. Your support makes the distribution possible.
